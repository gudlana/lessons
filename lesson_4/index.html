Анатомія CSS

Селектор - запит (пошук) елемента в HTML файлі до якого підключений CSS-файл

Синтаксис:

  • селектор {
    • css параметр;
    • css параметр;
    • css параметр;
    • ...
    • css параметр;
  • }

CSS параметр:

css властивість: значення властивості

Типи селекторів

Тип №0. Селектор зірочка (*)

Шукаємо (робимо запит), звертаємось до ВСІХ об'єктів на сторінці.

--- Приклад ---

  • * {
    • color: red;
  • }

Тип №1. Селектор тегу

Шукаємо, звертаємось (робимо запит) по назві тегу.

-- Приклад ---

Звертаємось/описуємо (шукаємо, робимо запит) до УСІХ тегів DIV на сторінці

  • div {
    • color: red;
  • }

Тип №2. Селектор класу

Шукаємо, звертаємось (робимо запит) по назві класу
Спочатку крапка, потім ім'я класу.

--- Приклад ---

Звертаємось/описуємо (шукаємо, робимо запит) до УСІХ тегів з класом block на сторінці

  • .block {
    • color: red;
  • }

Тип №3. Селектор ідентифікатору

Шукаємо, звертаємось (робимо запит) по назві ID
Спочатку # потім значення ID.

--- Приклад ---

Звертаємось/описуємо (шукаємо, робимо запит) ТІЛЬКИ до одного об'єкту з ID some-id

  • #some-id {
    • color: red;
  • }

Тип №4. Селектори атрибутів

Шукаємо, звертаємось (робимо запит) по комбінації назви атрибуту та його значення
Може використовуватись коли нам треба описати стилі для будь-якого елемента але з певними умовами та параметрами.

--- Приклад ---

Звертаємось/описуємо (шукаємо, робимо запит) до всіх елементів по комбінації назви атрибуту class та його значення example-attr

  • [class*="example-attr"] {
    • color: red;
  • }

Комбінації селекторів

--- Приклад ---

Усі DIV але лише з класом example

  • div.example{
    • color: blueviolet;
  • }

--- Приклад ---

Усі теги але лише з класами example та text ОДНОЧАСНО
НІЯКИХ ПРОБІЛІВ МІЖ КЛАСАМИ НЕ СТАВИМО!

  • .example.text{
    • color: green;
  • }

--- Приклад ---

Звертаємось відразу до кількох класів

  • .fourth-order-title,
  • .sub-item-note-other{
    • color: rgb(95, 0, 143);
  • }

перелік селекторів може бути різний, порядок не важливий

  • span,
  • .sub-item-note-other2{
    • color: red;
  • }

Каскадність (вкладеність) селекторів

Фактично змінюємо умову запиту до елементів сторінки
Уточнюємо що саме шукаємо
ОБОВ'ЯЗКОВО ПРОБІЛ МІЖ ЕЛЕМЕНТАМИ!

--- Приклад ---

Усі теги A але тільки в середені об'єктів з класом list-note
рівень вкладеності значення не має

  • .list-note a{
    • color: chocolate;
  • }

--- Приклад ---

Усі об'єкти з класом black-text в середені об'єктів з класом list-note
рівень вкладеності значення не має

  • .list-note .black-text{
    • color: black;
  • }

--- Приклад ---

Усі об'єкти в середені об'єктів з класом example-casc

  • .example-casc *{
    • color: darkgreen;
  • }

Пріоритет селекторів

Основні правила формування пріоритетів

  1. У випадку рівнозначних селекторів більший пріоритет буде у останнього (нижнього) по коду
  2. Більший пріоритет завжди буде у селектора, який звужує (уточнює) пошук (запит)
  3. Пріоритет у рядкових (inline) стилів (атрибут style="") вищий ніж у стилів з окремого файлу (наприклад style.css)
  4. Флаг !important перезаписує пріоритет (не рекомендується)
  5. Селектор класу і селектор атрибуту рівнозначні (працює правило №1)
